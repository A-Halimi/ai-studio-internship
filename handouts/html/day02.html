2 Field guide the concepts

Supervised learning means learning from labeled examples. Show a machine hundreds of (X, y) pairs — features in, correct label out — and it learns the mapping, then guesses the label for a brand-new example. But how do you know it really learned, versus just memorized? You hide part of the data. Split into a training set (it studies these) and a test set (locked in a drawer), and grade only on the test set. It's the difference between studying past papers and sitting a fresh exam.

Two classifiers. k-NN labels a new penguin by a majority vote of its k nearest neighbors — small k memorizes (overfits), large k generalizes. A decision tree writes itself a flowchart of yes/no questions you can read out loud. And one trap: k-NN measures "nearest" by distance, so a big-numbered feature like body mass can drown out the rest — StandardScaler gives every feature a fair vote.

3 Lab missions check each off in the notebook

  1. ☐Split the data honestly train_test_split(..., test_size=0.25, stratify=y). Expect: 249 training penguins, 84 held out for testing.
  2. ☐Build k-NN from scratch Fill in knn_predict: distance to all points → k nearest → majority vote. Expect: it labels the first test penguin correctly.
  3. ☐Grade it on the test set Run your k-NN on every test penguin and measure accuracy. Expect: around 0.95 (the self-check insists on beating 0.80).
  4. ☐The same thing in 3 lines KNeighborsClassifier — .fit then .score. Expect: almost exactly your from-scratch score. Same algorithm.
  5. ☐Grow a tree DecisionTreeClassifier(max_depth=3), then plot it. Expect: a shallow flowchart you can trace as a sentence.
  6. ☐Build the confusion matrix confusion_matrix(y_test, y_pred) as a heatmap. Expect: a bright diagonal; the few mix-ups cluster on Chinstrap.
  7. ☐Run the tournament (raw features) Try every k from 1–15 on all four unscaled features. Expect: four raw features score worse than two — the trap.
  8. ☐Scale, then rerun Same tournament on StandardScaler-scaled features. Expect: accuracy jumps above 0.95 at the best k.

6 Mini-glossary

supervised learning
Learning a mapping from labeled examples (features → known answer).
train/test split
Hiding part of the data to grade the model honestly on unseen examples.
k-NN
Classify a point by the majority label of its k nearest neighbors.
decision tree
A learned flowchart of yes/no questions ending in a prediction.
accuracy
The fraction of predictions that are correct.
confusion matrix
A grid of true vs predicted labels — shows where mistakes happen.
overfitting
Memorizing the training data (noise and all) instead of the general pattern.
