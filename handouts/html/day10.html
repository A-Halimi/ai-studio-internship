2 Field guide the three acts

Act I — Assemble. Every notebook exported its model files into ai_studio/models/. Launch the Studio and each satisfied module unlocks its tab. Ten files, seven modules, one app.

Act II — Present. A demo is a science talk: hook → two favorite modules live → your hardest bug → what surprised you. A good model card states honestly what a model does, how accurate it is, and where it fails. Honesty is the science.

Act III — the Grand Benchmark. Multiply two big matrices three ways — a hand-written triple loop, NumPy, and the GPU — then time them and plot the result on log-log axes. The cost grows like O(n³), and the GPU comes out ~10,000× faster than the loop.

🧮 Matrix Watch — the payoff. Every conv, every attention head, every Q-update you built runs on C = A @ B. Matrix multiplication was the whole game all along — and Weeks 3–4 (Julia / HPC) make it scream. Bring this plot.

4 Cheat sheet benchmarking matmul

for i,j,k: C[i,j] += A[i,k]*B[k,j]Matmul by definition — the O(n³) triple loop.
t = time.perf_counter()Read a high-resolution clock (take best of 3).
torch.cuda.synchronize()Wait for the async GPU before you stop the clock.
gflops = 2*n**3 / t / 1e9Turn seconds into GFLOPs (a matmul is 2n³ ops).
C = A @ BThe one line every Studio module secretly runs.

6 Mini-glossary

benchmark
A controlled speed test comparing methods fairly.
FLOP / GFLOPs
A floating-point operation · billions of them per second.
Big-O / O(n³)
How cost grows with size; matmul scales as n cubed.
vectorization
Replacing Python loops with fast bulk array operations.
HPC
High-performance computing — big math, made fast at scale.
Julia
The fast scientific language you'll use in Weeks 3–4.
